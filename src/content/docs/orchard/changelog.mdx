The change log is Orchard's record of what's happened across your stack — a running
timeline of events and changes you can scroll back through. When something looks off on
the [dashboard](/orchard/dashboard/), this is where you find out what led up to it.

## What you'll find

- **Events** — notable activity across Bitcoin, Lightning, and the mint as it occurs.
- **Changes** — configuration and state changes, so you can see what changed and when.

<Aside type="caution" title="The event detail is still taking shape">
  Orchard is under active development, and which events are logged — and how you filter
  and read them — is still settling, so the specifics land here as they stabilize. The
  [Orchard repository](https://github.com/cashubtc/orchard) has the source code in the
  meantime.
</Aside>
